Agent Code 0.0.3 is available (you have 0.0.2).
Three changes since your version — including this very panel.

When a new version of Agent Code ships, every installed copy now checks GitHub Releases every 6 hours and offers you the new .vsix right inside VS Code. Click Update, a progress bar streams the download, VS Code installs it, and you reload the window. That's the whole loop.

The check is fail-soft: no network, no toast. And you can opt out anytime from agentCode.checkForUpdates.

For the curious — how it works
The extension polls repos/elyas-tirit/Agent-Code/releases/latest, parses the semver tag, and compares it to package.json version. State (last check, "skipped" version) lives in globalState. The .vsix is streamed via web fetch + Readable.fromWeb piped through a counting Transform into fs.createWriteStream — so the progress bar is real, not theatrical, and a 70 MB download won't allocate 70 MB of JS heap. Install goes through workbench.extensions.installExtension on a Uri.file.
⚡
Updates are 17× lighter
Bundled binaries you didn't need — gone.

Every previous build dragged a ~150 MB platform-specific claude binary inside the package — useless, because Agent Code uses your globally-installed Claude Code CLI. Dropping those native packages from the .vsix shrinks the install (and every future update) to about 9 MB.

In practice: updates now finish in seconds even on a slow connection, and the install footprint stops eating disk for no reason.

For the curious — what changed
Added node_modules/@anthropic-ai/claude-agent-sdk-*/** to .vscodeignore. The SDK still works because we pass pathToClaudeCodeExecutable explicitly from resolveClaudePath() — pointing at the user's installed CLI (/opt/homebrew/bin/claude, etc.), never the bundled copy. Also dropped CLAUDE.md (internal handoff) and .github/ from the package.
